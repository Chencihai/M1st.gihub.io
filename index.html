*高级日志分析的问题和答案*

作者：伊兹米尔 ·伊祖丁

1.问：网络安全分析师应监控的关键Windows 事件ID 是什么?它们表示什么?

回答：

  1. 事 件ID 4624: 表示登录成功。对此进行监控可以帮助跟踪用户身份验证以检测潜在 的未经授权的访问。

  2. 事件 ID 4625: 表示登录尝试失败。此事件多次发生可能表明存在暴力攻击。

  3. 事 件ID 4634: 当用户从会话注销时记录。它有助于跟踪会话持续时间并识别异 常活动。

  4. 事件ID 4688: 跟踪新进程的创建。这对于检测可能表明恶意软件执行的可疑进程 至关重要。

  5. 事 件ID 4697: 在系统上安装服务时记录。监控这一点可以帮助检测未经授权的服 务，这可能是后门。

  6. 事件 ID 4720:表示创建新用户帐户。分析师应调查此事件，以确保帐户创建获得 授权。

  7. 事件 ID 4740: 表示帐户已被锁定，可能是由于多次登录尝试失败，这可能意味着暴力 攻击。

  8. 事件 ID 4769: 与 Kerberos 服务票证请求相关。分析师对此进行监控，以检测潜在的 “票证传递”攻击。

  9. 事 件ID 4776: 表示凭据验证过程。重复发生失败的事件可能表明存在密码喷射攻 击。

10.事件ID 5140:监视对网络共享的访问。不寻常的访问模式可能表明网络内存在横 向移动。

  1. 问题：您如何分析来自多个IP 地址的一系列失败登录尝试?

回答：

  1. 聚合和关联日志：

0 收集与失败登录尝试相关的日志，按事件ID 4625进行过滤(适用于 Windows 环境)。

o 聚合这些日志以识别模式，例如常见的 IP地址、用户名或时间戳。

  1. 识别模式：

o 查找针对多个用户帐户的特定IP地址的重复失败尝试，这可能表明存在暴力 攻击或密码喷雾攻击。

o 检查尝试之间的时间间隔，以确定它们是自动还是手动。

  1. 地理位置分析：

o 将 IP 地址与GeolP 数据库交叉引用，以确定失败的尝试是否来自意外或外 部位置。

  1. 与成功登录对比：

o 检查在来自相同或不同 IP地址的一系列失败之后是否有任何成功的登录尝 试，这可能表明帐户已被盗用。

  1. 与其他日志的交叉引用：

o 调查其他日志源，例如防火墙或VPN 日志，以查看相同的IP 地址是否正在 尝试或以其他方式访问网络。

  1. 应对措施：

0 在防火墙处或通过WAF(Web 应用程序防火墙)阻止违规 IP地址。

o 强制重置目标帐户的密码，并考虑实施多重身份验证(MFA)。

0 监视受影响的系统并密切关注任何进一步的可疑活动。

  1. 问题：您将采取哪些步骤来调查防火墙日志中检测到的可疑出站网络流量?

回答：

  1. 识别流量：

0 隔离指示可疑出站流量的特定日志，重点关注异常目的地、端口或数据量。

分析源码：

0 确定网络中流量的来源，包括生成流量的设备或服务器。

0 检查负责源设备上流量的进程或应用程序。

  1. 结合交通情况：

0 确定相关设备或应用程序的流量是否正常，或者是否异常。

o 与其他日志(例如DNS 日志)交叉引用，以识别流量所针对的域名或IP 地 址。

  1. 调查目的地：

0 研究目标 IP地址或域，以确定它们是否是已知的恶意实体或公司使用的外部 服务的一部分。

o 使用威胁情报源检查IP地址或域是否已被标记为存在恶意活动。

  1. 检查妥协指标：

0 查看端点日志，例如事件ID 4688和4697(进程创建和服务安装),以确定 是否涉及恶意软件。

o 对已知的 loC (妥协指标)执行快速扫描，例如与流量关联的特定哈希值 或注册表项。

  1. 减轻：

0 如果流量被认为是恶意的，请在防火墙处阻止它并隔离受影响的设备。

o 通知相关利益相关者并根据事件响应计划升级。

进一步监测：

0 继续监控网络是否有任何其他可疑活动或来自其他设备的流量模式。

0 更新防火墙规则和IPS/IDS 签名以检测并阻止将来的类似活动。

  1. 问题：如何使用日志数据识别和响应网络内的横向移动?

回答：

  1. 检测：

0 通过监视网络内多台计算机上的事件ID (例如4624(登录)和4672(分配 的特殊权限))来识别横向移动。

o 查找不同系统的异常登录顺序，特别是如果它们发生在正常工作时间之外或 涉及服务帐户。

  1. 相关活动：

o 将登录事件与横向移动中通常使用的进程或工具(例如 wmic.exe、 PsExec或 PowerShell) 相关联。

0 检查是否存在对可用于访问其他系统的管理工具或命令的可疑使用。

研究登录类型：

0 请注意登录类型3(网络)和10(远程交互),这可能表示尝试使用网络共 享或RDP 会话进行横向移动。

  1. 分析网络流量：

o 使用网络日志来识别通常不进行通信的系统之间异常的 SMB、RDP 或 RPC 流量模式。

o 查找凭证被盗或重复使用的证据，例如来自不同计算机的重复4769 (Kerberos 服务票证请求)事件。

  1. 遏制：

0 隔离显示横向移动证据的系统，以防止进一步传播。

o 重置密码或禁用已用于横向移动的帐户，尤其是特权帐户。

  1. 根除：

o 识别并删除用于横向移动的任何恶意软件或工具，例如远程访问工具或 Mimikatz。

o 修补任何允许横向移动的漏洞，并加强网络分段以限制访问。

  1. 事件后行动：

0 通过使用可触发可疑模式警报的SIEM 关联规则，增强对横向移动的早期 检测的监控。

o 教育用户和管理员了解横向移动的风险并强制执行最小权限原则。

  1. 问题：如何在日志中区分合法和恶意的PowerShell 执行?

回答：

  1. 分析命令行参数：

0 查看PowerShell 执行的完整命令行参数。恶意命令通常包括编码的有效负 载(EncodedCommand)、 执行策略绕过(-ExecutionPolicy Bypass)或可疑 网络连接。

  1. 检查是否混淆：

0 查找混淆迹象，例如大量使用Base64 编码的命令、随机字符插入或重复使用 串联(+)。合法脚本通常不采用此类方法。

  1. 回顾一下上下文：

o 检查启动PowerShell 的父进程。如果父进程是意外的东西，例如 explorer.exe或 winword.exe, 则可能很可疑。

0 检查PowerShell 进程是否由用户操作(例如打开文档或单击链接)触发，或 者是否以自动或隐藏方式生成。

  1. 检查执行频率：

o 评估脚本运行的频率。与合法任务不相关的重复或计划的 PowerShell 执行 值得调查。

  1. 与威胁情报的交叉参考：

o 将脚本的行为与来自威胁情报源的已知恶意技术或loC 进行比较。许多攻击重 复使用技术，可以使用更新的情报来发现这些技术。

  1. 评估权限和影响：

o 检查PowerShell 脚本是否尝试使用提升的权限执行或修改关键系统设置。在 没有明确原因的情况下执行此操作的脚本通常是恶意的。

  1. 调查网络活动：

o 如果PowerShell 命令涉及网络活动，例如下载文件或连接到远程服务 器，则这是一个危险信号。

合法脚本很少进行出站连接。

  1. 回复：

o 如果是恶意的，请立即终止PowerShell 进程并隔离受影响的系统。

o 捕获完整的命令行以进行进一步的取证分析并创建针对未来类似威胁的检测规 则。

6.问：如何使用日志数据检测和分析潜在的数据泄露事件?

回答：

  1. 识别异常流量：

0 监控大量数据传输的出站网络流量，尤其是陌生或外国IP 地址的出站网络 流量。为此，请使用防火墙、代理和DLP (数据丢失防护)日志。

  1. 分 析DNS 查询：

0 调查异常的DNS 查询，特别是最近注册的域或已知托管恶意内容的域。 重复查询与典型业务活动不匹配的外部站点可能是渗透的迹象。

  1. 检查可疑文件访问：

0 查看文件访问日志，以确定敏感数据是否被通常不需要访问该信息的用户访 问或修改。

注意来自文件服务器的大量访问或传输。

  1. 与端点日志关联：

0 将端点日志(例如，事件ID 5145,记录通过SMB 的文件访问)与网络日志进 行交叉引用，以确定敏感文件是否被复制到外部设备或上传到互联网。

  1. 检查进程日志：

o 调查负责文件传输的进程。查找是否使用了未经授权或不常见的工具，例如 WinSCP、curl 或员工通常不使用的自定义脚本。

  1. 检查异常的身份验证模式：

o 确定参与数据访问的用户帐户是否有异常登录，例如来自意外位置或使用不同 设备。这可能表明帐户已被盗用。

  1. 应对和缓解措施：

o 如果确认发生泄露，请立即阻止进一步的数据传输，隔离受影响的系统， 并撤销受感染帐户的访问权限。

o 与法律和合规团队合作，了解影响并准备必要的通知。

0 实施更严格的数据监控和审查访问控制，以防止未来发生事件。

  1. 问题：您将采取哪些步骤来分析PowerShell 日志记录(事件ID 4104) 以检测潜在的 恶意活动?

回答：

  1. 启用详细的 PowerShell 日志记录：

0 确保 PowerShell 日志记录配置为捕获脚本块(事件ID 4104),这可以 提供对正在执行的命令的可见性。

  1. 检查可疑命令：

o 寻找潜在恶意行为的迹象，例如：

使 用Invoke-WebRequest 或 Invoke-Expression 下载或执行远程代 码。

§编码命令(-EncodedCommand) 通常隐藏脚本的意图。

§ 使 用 -ExecutionPolicy Bypass绕过执行策略。

§不寻常的脚本块内容，例如禁用安全工具、建立持久性或泄露数据 的命令。

  1. 检查是否混淆：

0 通过查找随机字符串或分成多个串联部分的命令来识别混淆代码。合法的脚 本很少使用这些技术。

  1. 与网络活动的相关性：

o 将 PowerShell 活动与网络日志交叉引用，以查看脚本是否正在建立外部连 接，尤其是与已知的恶意域或IP 地址的连接。

  1. 分析执行上下文：

o 确定执行脚本的用户帐户和计算机。在特权帐户下或关键服务器上运行的脚本 需要更严格的审查。

  1. 自动检测：

o 使用SIEM或 EDR 工具自动检测常见恶意 PowerShell 模式。为指示威胁的 特定关键字和模式创建警报。

  1. 应对措施：

0 5 如果检测到恶意活动，请隔离受影响的系统、阻止出站网络连接并分析完整的 PowerShell会话日志以了解危害的程度。

  1. 问题：如何使用日志数据识别内部枢轴攻击?

    回答：

    1.监控横向运动指标：

0 使用相同的凭据跟踪不同系统上的多次成功登录尝试，尤其是在正常行为模式 之外。4624(登录)和4648(使用显式凭据登录)等事件 ID 是关键指标。

  1. 分析网络流量：

0 查找通常不通信的系统之间的异常内部流量。具体来说，监视 SMB、RDP 和 WMI 连接，这些连接通常用于横向移动。

  1. 调查管理工具的使用：

o 检查日志以了解使用PsExec、wmic.exe和 net.exe 等管理工具进行横向移 动的情况。这些工具生成特定的日志条目(例如，用于进程创建的事件 ID 4688)

  1. 与安全日志的关联：

o 将系统日志与其他安全日志(例如来自防火墙或 IDS/IPS的日志)关联 起来，以确定内部枢纽是否是更广泛攻击的一部分。

  1. 端点日志：

0 查看端点日志，以确定防火墙规则、用户组或未经授权的软件安装是否有任 何更改，这可能会促进进一步的横向移动。

  1. 应对和遏制：

o 如果确认发生内部枢纽攻击，请立即隔离受影响的系统、轮换凭证，并上报给 事件响应团队以进一步遏制。

  1. 攻击后分析：

0 遏制后，进行详细分析，了解攻击者如何在内部进行转移，识别所有受影响 的系统，并确定攻击的入口点。

  1. 问题：日志中的哪些指标可能表明存在权限提升攻击?

回答：

  1. 异常账户活动：

o 查找突然执行需要提升权限的操作的低特权帐户，例如访问敏感文件或安装软 件。相关事件ID 包括4672(分配特殊权限)和4624(登录)。

  1. 检查令牌操作的安全日志：

0 监视日志以发现令牌操纵攻击，其中攻击者试图冒充其他用户或使用 mimikatz 等工具提升权限。

查找事件 ID 4673 (调用了特权服务)。

  1. 分析计划任务和服务：

o 检查是否创建了可用于以提升的权限运行代码的计划任务或新服务。事件 ID 4697 记录新服务的安装。

  1. 审核流程创建日志：

0 调查日志中通常与权限升级相关的进程，例如以高完整性级别运行的cmd.exe 或 powershell.exe。事件 ID 4688在这里很有用。

  1. 查找文件权限更改：

0 查看文件系统日志以了解文件或目录权限的更改，这些更改可能允许非特权用 户以提升的权限修改或执行文件。

  1. 网络异常：

o 识别权限升级后的任何横向移动。攻击者通常会在横向移动以获得对更关键系 统的控制权之前升级权限。

  1. 立即响应：

o 如果检测到权限升级，请撤销受感染帐户的访问权限，隔离受影响的系统， 并进行彻底调查以了解升级方法。

10.问题：如何使用日志分析检测命令和控制(C2)通信?

回答：

  1. 识别异常目的地的出站流量：

0 监视与陌生IP地址或域的出站连接，尤其是那些与已知业务需求不符的连接。 重点关注罕见或长期存在的出站会话。

  1. 检查信标模式：

0 分析网络日志以查找到同一IP地址或域的定期出站连接，这是典型的C2 信 标行为。

  1. 检查协议使用情况

0 查找出站流量中使用的不常见协议，例如DNS 隧道、非标准端口上的 HTTP/HTTPS流量或与已知业务应用程序不相符的加密流量。

  1. 与其他指标的相关性：

0 将网络流量与来自其他安全控制(例如IDS/IPS 或端点检测工具)的日志进行 交叉引用，这些工具可能会检测恶意负载或命令执行尝试。

  1. 调查异常用户代理字符串：

0 识别 HTTP 流量中不寻常或罕见的用户代理字符串，这可以指示与C2 服务器 的通信，特别是当它们偏离典型的组织模式时。

6.DNS 日志分析：

0 查找对动态生成域(DGA) 或异常长域名的DNS 查询。C2 服务器经常使 用这些来逃避检测。

  1. 对检测到的C2 活动的响应：

0 如果检测到C2 通信，请立即阻止到已识别的 IP/域的出站流量，调查受影响的 系统，并禁用任何受损的帐户或后门。

11.问题：如何分析日志以检测对RDP (远程桌面协议)服务的暴力攻击?

回答：

  1. 监控多次失败的登录尝试：

o 查Windows 安全事件日志以了解重复的失败登录尝试(事件 ID 4625)。单 个IP地址在短时间内出现大量失败可能表明存在暴力尝试。

  1. 检查失败后登录是否成功：

o 在一系列失败的尝试之后，查找成功的登录(事件ID 4624)。这种模式通 常表明攻击者最终猜出了正确的凭据。

  1. 识别异常源IP地址：

0 检查登录尝试的源IP。重点关注来自国外或通常不访问系统的IP地址。

如果可能的话，与地理位置数据相关联。

  1. 分析账户锁定事件：

0 查看帐户锁定事件(事件ID 4740)以识别暴力攻击的目标帐户。频繁 的锁定可能会扰乱合法用户并发出持续攻击的信号。

  1. 与防火墙日志的关联：

0 将失败的登录尝试与防火墙日志关联起来，以确定同一IP 地址是否正在尝试访 问多个服务。这可能表明您的网络遭受了更广泛的攻击。

  1. 使用 SIEM自动检测：

0 在 SIEM 中设置警报，以便在特定时间范围内达到登录尝试失败阈值并 成功登录时触发。

  1. 应对措施：

o 如果确认存在暴力攻击，请阻止违规IP地址、强制执行帐户锁定策略、实施多 重身份验证(MFA), 并确保所有帐户都拥有强大且复杂的密码。

12.问题：您会在日志中查找什么内容来检测攻击者对 live-oU-the-land 二进制文件(LOLBins)的使用? 回答：

  1. 监视系统实用程序的执行：

0 查看日志以了解攻击者经常滥用的系统实用程序的执行情况，例如 rundll32.exe、powershell.exe、certutil.exe、wmic.exe 和 regsvr32.exe。事件 ID 4688 (进程创建)在这里特别有用。

  1. 识别不寻常的命令行参数：
